# 虾得乐网站视觉改版需求文档 V2

## 1. 改版目标

当前网站整体风格过于偏深色、偏硬核、偏技术型 demo，不符合虾得乐现在希望传达的产品气质。

这次改版需要把整体视觉方向调整为：

**更轻松、更清新、更有幽默感、更有角色记忆点。**

新版本不是要做成严肃的技术展示站，而是要更像一个：

- 面向工作场所的轻松产品展示页
- 有一点可爱感和故事感的路演网站
- 能承载图片、动画、插画和角色感的页面
- 适合展示“给工作场所里 i 人的一点低压力娱乐”这个核心概念

整体需要让用户感受到：

- 虾得乐是友好的
- 虾得乐是有点好玩的
- 虾得乐不是冷冰冰的 bot
- 虾得乐像一个会在群里陪大家轻松一下的角色

---

## 2. 新的视觉关键词

请全站统一围绕以下关键词设计：

- 清新
- 轻松
- 幽默
- 可爱但不幼稚
- 友好
- 工作场所里的轻娱乐
- 有角色感
- 适合放插画和动画
- 有留白
- 有轻微 playful 感
- 像一个聪明、讨喜、会接话的 bot

避免以下方向：

- 深色赛博科技风
- 冷硬系统面板感
- 重度发光和粒子流
- 过强未来感
- 过度游戏化
- 太像黑客/AI 工具官网
- 太像企业官网模板

---

## 3. 新配色方案

请整站改为明亮、柔和、空气感更强的浅色系风格。

### 主配色

- 背景奶白：`#FFFDF8`
- 浅薄荷绿：`#EAF8F3`
- 浅天蓝：`#EAF4FF`
- 标题深灰蓝：`#2F3A4F`
- 正文灰：`#6E7A8F`
- 主按钮蓝：`#7CB7FF`
- 辅助按钮绿：`#8DDFC3`
- 点缀橙黄：`#FFC97A`
- 卡片白：`#FFFFFF`

### 使用建议

#### 页面背景
- 页面主背景以 `#FFFDF8` 为主
- 区块背景可交替使用 `#EAF8F3` 和 `#EAF4FF`
- 局部可加入柔和渐变，例如奶白到浅蓝、奶白到浅薄荷绿

#### 文字
- 大标题统一使用 `#2F3A4F`
- 正文统一使用 `#6E7A8F`
- 次级说明文字颜色可以稍浅，但仍需保证可读性

#### 按钮
- 主按钮使用 `#7CB7FF`
- 辅助按钮使用 `#8DDFC3`
- hover 可以略微加深或增加柔和阴影
- 可少量使用 `#FFC97A` 做角标或提醒点缀

#### 卡片
- 卡片底色统一为 `#FFFFFF`
- 使用浅阴影和浅描边，不使用深色玻璃拟态
- 边框建议为非常浅的蓝灰或透明白

---

## 4. 整体 UI 风格要求

## 4.1 页面气质

请把页面理解为：

**工作场所里的轻松搭子**
而不是
**未来感智能体系统**

整体 UI 需要更轻盈、更松弛、更有呼吸感。

希望用户看到页面时会觉得：

- 这个 bot 很亲近
- 这个产品不让人有压力
- 这不是严肃炫技，而是一个让工作群轻松一点的小角色

---

## 4.2 背景处理

不要再使用大面积纯深色背景。

建议采用以下背景层次：

- 奶白大底
- 局部浅蓝 / 浅绿渐变块
- 模糊彩色光斑
- 很淡的小圆点 / 小气泡 / 小装饰图形
- 留白更多
- 背景节奏轻，不要复杂

可以加入：

- 柔和的 radial gradient
- 大面积模糊圆形背景
- 像纸片、便签、贴纸一样的装饰元素
- 轻微的办公室桌面感

不要加入：

- 科技网格过强
- 硬核线框
- 激光、扫描、流动代码等视觉元素

---

## 4.3 卡片风格

所有 section 卡片都需要改成更柔和、更适合插画和图片的风格。

### 卡片建议
- 白色底
- 大圆角，建议 `24px` 到 `32px`
- 阴影轻柔
- 可加入轻微彩色描边
- 内边距更宽松
- 内容不要挤
- 要适合塞图片、gif、插画或留白

### 卡片视觉方向
可以参考：
- 贴纸
- polaroid
- 聊天气泡
- 便签卡片
- 桌面小组件

### 不要做成
- 黑色玻璃面板
- 硬朗边框科技板
- 强发光芯片面板

---

## 4.4 图标风格

继续使用 icon，不允许 emoji。

但 icon 的整体感觉也要变得更轻松，不要太硬。

建议：
- 线性 icon
- 粗细中等
- 配轻微彩色底板
- 可放在圆角小色块里
- 整体更像“轻产品展示”而不是“技术功能模块”

---

## 5. 动效改版方向

## 5.1 推荐保留的动效

请保留以下类型的轻动效：

1. 缓慢漂浮的小圆点、小星芒、小气泡
2. 轻微摇摆的小贴纸元素
3. 卡片 hover 时轻轻上浮
4. 按钮点击时的小扩散波纹
5. 页面空白处慢速漂浮的小装饰图形

这些动效要轻、慢、自然，不要喧宾夺主。

---

## 5.2 推荐新增的可爱动画

请加入以下类型的轻量动画元素：

1. **虾得乐小头像轻微呼吸感**
   - 可放在 Hero 区或某个角落
   - 动作可以是轻微放大缩小、轻轻上下浮动

2. **某个角落有会动的聊天气泡**
   - 可做成轻微浮动或弹出效果
   - 不用太频繁，保持可爱和克制

3. **剧本杀 / 卧底功能卡片有轻微翻牌感**
   - hover 时轻微倾斜、翻面或卡片位移
   - 不要做成真实 3D 重动画

4. **标题旁边有个小龙虾探头、眨眼、晃动**
   - 这是提升“虾得乐角色记忆点”的重点
   - 可以用 CSS 小动画、SVG 或简易插画实现
   - 动作要小，不要幼稚夸张

5. **页面空白处有慢速漂浮的小装饰图形**
   - 可以是星星、气泡、贴纸、圆片、聊天框小块
   - 风格统一，颜色柔和

---

## 5.3 不再保留的旧动效方向

请删弱或移除以下内容：

- 大面积粒子流
- 重赛博科技粒子背景
- 强 glow 发光边框
- 硬核扫描感
- 大量科技轨迹线
- 过强的沉浸式暗光动画
- 太像 AI 工具官网的动效氛围

---

## 6. 图片区与留白设计

这次改版需要明确预留更多自然的图片区和动画位，而不是只靠文字和规则矩形。

## 6.1 图片容器风格

图片区不要只是普通图片框，请改成更自然、可爱的形式，例如：

### A. Polaroid 卡片
适合放：
- 飞书聊天截图
- 现场照片
- 产品操作图

特点：
- 像拍立得照片
- 有边框和留白
- 可以轻微旋转
- hover 时轻轻摆动或上浮

### B. 贴纸板 / 拼贴板
适合放：
- 多张小图
- 功能截图组合
- 场景图组合

特点：
- 像几张图片被“贴”在页面上
- 有轻微错位和层叠
- 可搭配胶带、图钉、圆角贴纸效果

### C. 聊天气泡式图片区
适合放：
- 飞书群聊界面截图
- 机器人回复示意

特点：
- 让图片像从聊天中自然长出来
- 更贴合产品形态

### D. 白底展示窗
适合放：
- 技术目录截图
- 项目结构图
- 流程图

特点：
- 简洁、清楚
- 适合技术内容
- 不破坏整体轻松感

---

## 6.2 图片与动画留位原则

每个大 section 尽量都留出：
- 一块大图片区
- 或一块可以替换为动画 / 插画的区域

这些区域不要做得太死，要方便后续替换为：
- 飞书截图
- 群聊展示图
- 目录结构图
- 可爱小动画
- 插画素材
- 比赛现场图片

---

## 7. 页面结构上的视觉改法

## 7.1 Hero 首屏

Hero 不再是“技术发布页感”，而要更像一个有角色感的欢迎区。

### Hero 视觉要求
- 背景明亮
- 大标题更有亲和力
- 右侧或旁边留一个大角色位 / 插画位 / 图片位
- 可以放虾得乐小角色、聊天气泡、小动画
- 不要让首屏太满，要有呼吸感

### Hero 可以包含
- 虾得乐名字
- 一句轻松的核心表达
- 两个按钮
- 一块大图或角色区
- 周围点缀漂浮的小图形

### Hero 的期望感受
用户看到首屏时，不应该想到“系统架构”，而应该先感受到：

**这个 bot 好像真的挺适合待在群里。**

---

## 7.2 Problem / Solution 区

这两个 section 不要再太像严肃咨询汇报。

### 建议改法
- 每个痛点做成轻卡片
- 卡片顶部加 icon 或小插画
- 留白更大
- 语言更自然
- 可以穿插小图、小气泡、小贴纸

---

## 7.3 Features 区

Features 可以更 playful 一点。

### 建议做法
- 三张卡片风格不同但统一
- 卡片里可留动效位
- 剧本杀 / 卧底卡片可加轻微翻牌感
- 氛围搭子卡片可加聊天气泡动画

### 视觉目标
让用户感觉这些功能不是“堆出来的模块”，而是“不同的互动方式”。

---

## 7.4 Tech 区

Tech section 也要一起软化，不要再用深色科技语气。

### 视觉建议
- 技术图放在白底展示窗里
- 配色仍然统一成浅色系
- 可加小标题、小标签，但不要太硬核
- 可以通过“拼贴式目录图 + 短说明”来展现工程能力

### 气质要求
技术区也应该是：
- 可信
- 清楚
- 但不压人

---

## 7.5 CTA 区

CTA 区不要太像“现在点击体验”。

要更像：

**不如去群里看看虾得乐今天在干嘛。**

可以保留主按钮，但整体语气更轻一点。

---

## 8. 文案语气同步调整

既然 UI 改轻松了，文案语气也要同步变得更自然。

### 文案关键词
- 轻松
- 亲和
- 有一点幽默
- 不用太书面
- 不要过度技术化
- 适合配可爱视觉

### 文案风格方向
更像：
- 工作群里一个轻松的小产品介绍
- 有角色感的 bot landing page
- 不是严肃的系统证明

### 避免
- 过强术语
- 过强工程汇报感
- 太像比赛 PPT 原句直接搬网页

---

## 9. 组件层面的具体改法

建议同步调整以下组件样式：

### 按钮
- 更圆润
- 颜色更明亮
- hover 有轻微上浮和阴影
- 点击有柔和波纹扩散

### 卡片
- 更大圆角
- 内边距更松
- 可加入顶部彩色标签
- 支持图片区插槽

### 图片组件
- 增加 polaroid / 贴纸板 / 展示窗三种样式
- 支持倾斜、层叠、漂浮动效

### 装饰组件
新增：
- 漂浮小星芒
- 小聊天框
- 小圆点
- 虾得乐头像组件
- 小龙虾探头组件

---

## 10. 建议开发实现方向

可以通过以下方式实现新风格：

- 浅色分区背景
- 大量圆角
- 柔和阴影
- 小范围渐变
- CSS 浮动动画
- 轻微 rotation / translate 动画
- Framer Motion 实现 reveal 和 hover
- SVG / CSS 小角色动效
- 图片容器组件化，方便替换内容

---

## 11. 对开发的直接改版指令

请将网站整体 UI 和色调从原本的深色科技风，改为更清新、轻松、幽默、友好的浅色风格。整体视觉基于以下配色：

- 背景奶白：#FFFDF8
- 浅薄荷绿：#EAF8F3
- 浅天蓝：#EAF4FF
- 标题深灰蓝：#2F3A4F
- 正文灰：#6E7A8F
- 主按钮蓝：#7CB7FF
- 辅助按钮绿：#8DDFC3
- 点缀橙黄：#FFC97A
- 卡片白：#FFFFFF

页面不再强调冷硬的技术感，而要更像“工作场所里的轻松搭子”，更适合表达虾得乐这个 bot 角色。请保留产品展示页的完整度，但整体气质要更有亲和力、更适合放插画、动画和图片。

请重点加入以下轻量动画和装饰方向：
- 虾得乐小头像轻微呼吸感
- 某个角落有会动的聊天气泡
- 剧本杀 / 卧底功能卡片有轻微翻牌感
- 标题旁边偶尔有个小龙虾探头、眨眼、晃动
- 页面空白处有慢速漂浮的小装饰图形
- 缓慢漂浮的小圆点、小星芒、小气泡
- 轻微摇摆的小贴纸元素
- 卡片 hover 时轻轻上浮
- 按钮点击时的小扩散波纹

请为页面预留更多自然的图片区和动画位，不要只使用普通矩形占位。图片区可做成 polaroid 卡片、贴纸板、聊天气泡式图片区、白底展示窗等，方便后续放飞书截图、目录图、功能图和现场展示图片。

最终希望整站看起来更像一个聪明、可爱、会在工作群里陪大家轻松一下的 bot 展示页，而不是深色技术 demo 网站。

---

## 12. 改版后的最终目标

改版完成后，整站应该让人感受到：

- 虾得乐更有角色记忆点
- 网站更适合放图、放动画、放插画
- 视觉更清爽、更轻松
- 产品更亲近，而不是更遥远
- 更符合“给工作场所里 i 人的一点低压力娱乐”这条主线

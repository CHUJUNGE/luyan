# 虾得乐 路演展示网站开发需求文档

## 1. 项目背景

我们要做一个用于比赛路演和 GitHub 展示的 React 网站，展示一个运行在飞书群聊中的 OpenClaw Bot，名字叫 **虾得乐**，团队名叫 **一侠天開**。

这个网站不是传统意义上的完整官网，也不是功能百科页，而是一个 **路演导向的 storytelling 展示网站**。它的核心目标不是讲全所有功能，而是用很短的路径让评委或观众快速明白三件事：

1. **工作场所里的群聊存在什么问题**
2. **虾得乐解决了什么痛点**
3. **这不是概念，而是已经接入飞书、可真实运行的 OpenClaw Bot，可以直接现场进入群聊展示**

同时，网站还需要有一个单独的 **答辩背景板页面**，用于现场问答时挂在屏幕上，展示产品图、飞书聊天截图、技术目录结构图等内容。

---

## 2. 项目定位

### 产品定位
**虾得乐是一个运行在飞书群聊中的 OpenClaw Bot，为工作场所里的 i 人提供低压力娱乐。**

### 网站定位
这是一个：
- 用于比赛路演的展示型网站
- 用于 GitHub Pages 部署的产品展示页
- 用于现场演示前承接叙事、演示后承接答辩的视觉载体

### 不要做成什么
不要做成：
- 传统企业官网
- 纯技术文档站
- 过于冗长的功能说明页
- 纯娱乐产品官网
- 游戏风过重的炫技页面

---

## 3. 核心叙事

整个网站要围绕下面这条主线展开：

> 在工作场所中，群聊往往承担通知、协作、同步信息的功能，因此容易高度工具化。  
> 想参与的人不一定愿意主动开口，想活跃气氛的人也不可能一直承担热场角色。  
> 尤其对 i 人来说，很多时候不是不想加入，而是不想进入高压社交。  
> 虾得乐，就是我们为工作场所里的 i 人提供的一种低压力娱乐。  
> 它运行在飞书群聊中，用更轻量、更自然的方式接住冷场、发起互动、推进小游戏，让参与比主动社交更容易。  
> 同时，它不是一个概念原型，而是一个已经搭建完成、具备工程结构、可在飞书群聊中真实运行的 OpenClaw Bot。

网站需要重点突出以下三个层次：

### 第一层：痛点
工作群过于工具化，轻娱乐和自然互动稀缺。

### 第二层：解决方案
虾得乐以低门槛、低压力、轻互动的方式让工作群重新有温度和节奏。

### 第三层：技术可信度
虾得乐不是简单回复脚本，而是一个已经具备模块化工程结构、可运行、可扩展的 OpenClaw Bot。

### 8.2 痛点区 Problem Section

#### 目标
明确告诉观众：我们不是做“一个好玩的机器人”，而是在解决真实的工作场所问题。

#### 结构建议
做成 3 张卡片或 3 个横向 block。

#### 三个核心痛点
1. **工作群太功能化**
   群聊主要围绕通知、协作、信息同步展开，一旦脱离任务本身，聊天就容易停住。

2. **活跃气氛依赖少数人**
   接话、热场、组织互动这类行为通常由少数外向者承担，但这件事很难持续。

3. **i 人想参与，但不想高压社交**
   不是不想加入，而是不想主动开口、不想被点名、不想承担带节奏的压力。

#### 区块标题建议
**我们想解决的，不是“没人说话”，而是工作群里参与成本过高。**

#### 区块底部总结句
**所以我们真正要做的，不是更吵的群聊，而是更容易参与的群聊。**

---

### 8.3 解决方案区 Solution Section

#### 目标
承接痛点，说明虾得乐如何介入工作场所群聊。

#### 标题建议
**虾得乐，如何把“主动社交”变成“自然参与”**

#### 结构建议
分成三条解决逻辑，不急着讲具体 feature 名字，先讲方案逻辑。

#### 三条方案逻辑
1. **低门槛接入**
   直接在飞书群里使用，不需要切换平台，不需要重新建场，也不需要学习额外工具。

2. **低压力参与**
   用户不需要成为气氛担当，也可以被自然带入互动流程。

3. **轻娱乐持续供给**
   不是一次性的热闹，而是可持续、可切换、可组合的互动机制。

#### 表达重点
这一部分要突出：
- 虾得乐不打断工作流
- 它嵌入的是现有工作场所基础设施，也就是飞书群聊
- 它提供的是“轻松加入的机会”，而不是强迫社交

Problem

我们想解决的，不是“没人说话”，而是工作群里参与成本过高。

工作群太功能化
群聊主要服务于通知、协作和同步，一旦离开任务本身，互动就容易停住。

活跃气氛依赖少数人
热场、接话、组织活动这些事，通常由少数外向者承担，但很难长期持续。

i 人想参与，但不想高压社交
很多人并不是不想加入，而是不想主动开口、不想被点名，也不想承担带节奏的压力。

总结句：
所以我们真正要做的，不是更吵的群聊，而是更容易参与的群聊。

Solution

虾得乐，如何把“主动社交”变成“自然参与”

低门槛接入
直接在飞书群里使用，不切换平台，不打断现有协作场景。

低压力参与
用户不需要成为气氛担当，也能被自然带入互动。

轻娱乐持续供给
通过可切换、可扩展的互动机制，让轻松的群聊娱乐持续发生。

Features

群聊氛围搭子
持续接话、定时活跃、承接情绪，让群聊不再轻易冷场。

AI 剧本杀 DM
自动推进剧情、抛出线索、维持节奏，把群聊升级成轻剧情现场。

谁是卧底 DM
自动发词、辅助投票、推进流程，让轻游戏互动在群里随时开始。

请基于以上需求开发一个 React + TypeScript + Vite 的路演展示网站。
